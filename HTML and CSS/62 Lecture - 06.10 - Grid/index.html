Grid

We use display of Grid, then we want to display elements inside of that grid parent element with alot of control over children elements positions.
But first we need to create the grid by setting up grid-columns and grid-rows. And after that we tell each children element what posistions they are taking inside of that grid.
Grid got quite a few different properties for parent (container) and children (items) elements. Also got a few new units and functions.

Grid Properties

Parent

  • display: grid;
  • grid-template-columns
  • grid-template-rows
  • grid-template: grid-template-rows / grid-template-columns;
  • grid-template-areas
  • row-gap
  • column-gap
  • gap: row-gap column-gap;
  • justify-content
  • justify-items
  • align-content
  • align-items
  • place-items: align-items justify-items;
  • place-content: align-content justify-content;
  • grid-auto-columns
  • grid-auto-rows
  • grid-auto-flow
  • grid: combination of 6 grid styles...

Children

  • grid-row-start
  • grid-column-start
  • grid-row-end
  • grid-column-end
  • grid-row: start / end
  • grid-column: start / end
  • grid-area: row-start / column-start / row-end / column-end
  • align-self
  • justify-self
  • place-self

Units and functions

  • fr - fractions
  • repeat(howMany, ofWhat)
  • span
  • min-content
  • max-content
  • auto
  • fit-content()
  • minmax()
  • min()
  • max()
  • masonry()
  • subgrid()

Grid vs Flex

Main idea is to use Flex for one dimensional layouts (either just column or just row) and use Grid for two dimensional layouts (controlling how elements inside of that grid are positioned).
